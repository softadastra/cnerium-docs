# Cnerium 0.5.0

Cnerium `0.5.0` is the first complete documented release of the Cnerium web framework stack.

This release defines the full framework layer from JSON parsing to the high-level `App` API.

It includes:

```txt
JSON
HTTP
Router
Middleware
Server
Runtime
App
```

The recommended entry point is:

```cpp
#include <cnerium/app/app.hpp>
```

For most projects, install:

```bash
vix add cnerium/app
```

## Version

```txt
0.5.0
```

## Status

```txt
early framework release
```

The API is usable, but the framework can still evolve quickly.

## App layer

The `cnerium::app` module is the recommended high-level API.

Example:

```cpp
#include <cnerium/app/app.hpp>
#include <vix/console.hpp>

using namespace cnerium::app;

int main()
{
  App app;

  app.get("/", [](AppContext &ctx)
  {
    ctx.text("Hello from Cnerium");
  });

  app.listen("127.0.0.1", 8080, []()
  {
    vix::console.info("Cnerium app is ready");
  });
}
```

The App layer connects:

```txt
runtime
server
middleware
router
HTTP
JSON
```

## AppContext

`AppContext` is the high-level request/response context passed to route handlers.

Example:

```cpp
app.get("/users/:id", [](AppContext &ctx)
{
  ctx.json({
    {"ok", true},
    {"id", std::string(ctx.param("id"))}
```

## Routing

This release supports route registration through the App API.

```cpp
app.get("/", handler);
app.post("/users", handler);
app.put("/users/:id", handler);
app.delete_("/users/:id", handler);
app.route(cnerium::http::Method::Patch, "/users/:id", handler);
```

Route parameters use `:name`.

```cpp
app.get("/shops/:shop_id/products/:product_id", [](AppContext &ctx)
{
  ctx.json({
    {"shop_id", std::string(ctx.param("shop_id"))},
    {"product_id", std::string(ctx.param("product_id"))}
  });
});
```

## JSON support

Cnerium `0.5.0` includes JSON support through `cnerium/json`.

Example JSON response:

```cpp
app.get("/health", [](AppContext &ctx)
{
  ctx.json({
    {"ok", true},
    {"status", "healthy"}
  });
});
```

Example JSON request body:

```cpp
app.post("/users", [](AppContext &ctx)
{
  auto body = ctx.json();

  ctx.status(cnerium::http::Status::created).json({
    {"ok", true},
    {"name", body["name"]}
  });
});
```

## Middleware

The middleware system allows global request/response logic.

Example:

```cpp
app.use([](auto &ctx, auto next)
{
  ctx.response().set_header("X-App", "Cnerium");
  next();
});
```

Middleware can:

```txt
inspect requests
add headers
log requests
protect routes
stop requests early
return JSON errors
run before and after handlers
```

## Runtime

The runtime provides concurrent task execution.

In app code, access it with:

```cpp
app.runtime()
```

Example:

```cpp
app.post("/jobs", [&app](AppContext &ctx)
{
  std::string path(ctx.path());

  app.runtime().post([path]()
  {
    vix::console.info("background job from", path);
  });

  ctx.status(cnerium::http::Status::created).json({
    {"ok", true},
    {"message", "job scheduled"}
  });
});
```

This release includes:

```txt
Task
ThreadPool
Executor
Scheduler
Runtime
ServerRunner
```

## Server layer

The lower-level server API is available through:

```cpp
#include <cnerium/server/server.hpp>
```

Example:

```cpp
#include <cnerium/server/server.hpp>
#include <vix/console.hpp>

using namespace cnerium::server;

int main()
{
  Server server;

  server.get("/", [](Context &ctx)
  {
    ctx.response().text("Hello from server");
  });

  vix::console.info("Server running on http://127.0.0.1:8080");

  server.run();
}
```

Most applications should use `App`.

Use `Server` when you need lower-level control.

## Error handling

Cnerium `0.5.0` supports custom error handlers.

```cpp
app.set_error_handler([](cnerium::server::Context &ctx,
                         const std::exception &ex)
{
  vix::console.error("unhandled exception:", ex.what());

  ctx.status(cnerium::http::Status::internal_server_error)
      .json({
        {"ok", false},
        {"error", "internal server error"}
      });
});
```

This keeps unexpected exceptions from escaping the request lifecycle.

## Not-found handling

Custom not-found handlers are supported.

```cpp
app.set_not_found_handler([](cnerium::server::Context &ctx)
{
  ctx.status(cnerium::http::Status::not_found)
      .json({
        {"ok", false},
        {"error", "route not found"},
        {"path", std::string(ctx.path())}
      });
});
```

This is useful for JSON APIs.

## Known notes

### APIs are still early

Cnerium is in early development.

The current API is designed to be simple and readable, but future versions may refine naming, handlers, runtime behavior, or server internals.

### Prefer App for normal projects

Use:

```cpp
#include <cnerium/app/app.hpp>
```

Avoid starting directly with lower-level modules unless you are testing internals.

### Use vix::console for logs

Application examples should use:

```cpp
vix::console.info(...)
vix::console.warn(...)
vix::console.error(...)
```

Use `vix::print` for standalone examples only.

### Copy request data before runtime tasks

Do not capture `AppContext&` in background tasks.

Wrong:

```cpp
app.runtime().post([&ctx]()
{
  vix::console.info(ctx.path());
});
```

Correct:

```cpp
std::string path(ctx.path());

app.runtime().post([path]()
{
  vix::console.info(path);
});
```

## Complete release example

```cpp
#include <cnerium/app/app.hpp>
#include <cnerium/http/Status.hpp>
#include <vix/console.hpp>

#include <string>

using namespace cnerium::app;

int main()
{
  AppConfig config;

  config.host = "127.0.0.1";
  config.port = 8080;
  config.thread_count = 4;

  App app(config);

  app.use([](auto &ctx, auto next)
  {
    ctx.response().set_header("X-App", "Cnerium");
    next();
  });

  app.get("/", [](AppContext &ctx)
  {
    ctx.json({
      {"ok", true},
      {"framework", "Cnerium"},
      {"version", "0.5.0"}
    });
  });

  app.get("/health", [](AppContext &ctx)
  {
    ctx.json({
      {"ok", true},
      {"status", "healthy"}
    });
  });

  app.get("/users/:id", [](AppContext &ctx)
  {
    ctx.json({
      {"ok", true},
      {"id", std::string(ctx.param("id"))}
    });
  });

  app.post("/jobs", [&app](AppContext &ctx)
  {
    std::string path(ctx.path());

    app.runtime().post([path]()
    {
      vix::console.info("background job from", path);
    });

    ctx.status(cnerium::http::Status::created).json({
      {"ok", true},
      {"message", "job scheduled"}
    });
  });

  app.set_not_found_handler([](cnerium::server::Context &ctx)
  {
    ctx.status(cnerium::http::Status::not_found)
        .json({
          {"ok", false},
          {"error", "route not found"},
          {"path", std::string(ctx.path())}
        });
  });

  app.set_error_handler([](cnerium::server::Context &ctx,
                           const std::exception &ex)
  {
    vix::console.error("unhandled exception:", ex.what());

    ctx.status(cnerium::http::Status::internal_server_error)
        .json({
          {"ok", false},
          {"error", "internal server error"}
        });
  });

  app.listen([]()
  {
    vix::console.info("Cnerium 0.5.0 example is ready");
  });
}
```
